# sre-agent-gitops
## 问题集锦一：网络配置问题
1.k8s集群中的Pod拉取镜像时需要访问外网的镜像仓库，这往往会被墙拦截。
2.想要解决该问题，一种方法是配置加速器或国内镜像仓库，但这对于自动化部署的管理将变得十分麻烦，也不符合企业的实际流程，因此我们通过代理解决无法访问问题。
3.但是由于集群中容器网络隔离，在使用奈云的Tun模式时很容易出现把容器当作Windows宿主机资源分配同一个适配器下主机号，使其无法与网桥网段和iptables规则兼容，使得它无法访问外部网络
4.使用以下代码修改容器内的http代理，可以解决这个兼容性问题，同时因为Pod作为集群资源是ansible控制的，所以不用担心Pod重启后访问不通的问题
```
kubectl edit deployment argocd-repo-server -n argocd
```
